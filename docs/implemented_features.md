# 구현 기능 문서

이 문서는 현재 코드베이스 기준으로 실제 구현된 기능을 정리한 문서입니다.

## 1. 앱 개요

이 앱은 임대 계약 정보를 등록하고, 만료 일정과 후속 연락 일정을 관리하는 Flutter 기반 임대 관리 앱입니다.

핵심 구성은 다음과 같습니다.

- 홈 대시보드
- 계약 목록 조회
- 계약 상세 관리
- 계약 등록 및 수정
- 로컬 저장
- 로컬 알림

## 2. 현재 구현된 화면

### 2.1 홈 화면

홈 화면에서는 계약 현황을 한눈에 볼 수 있습니다.

- 오늘 후속 연락이 필요한 계약 수 표시
- 현재 협의 중인 계약 수 표시
- 이번 달 만료 예정 계약 수 표시
- 오늘 후속 연락 목록 상위 3건 표시
- 만료 임박 계약 상위 3건 표시
- 전체 계약 보기 화면으로 이동
- 새 계약 등록 화면으로 이동

요약 카드를 누르면 해당 조건으로 필터링된 계약 목록 화면으로 이동합니다.

### 2.2 전체 계약 목록 화면

전체 계약 또는 특정 조건으로 필터링된 계약 목록을 확인할 수 있습니다.

- 계약 종료일 기준 오름차순 정렬
- 계약별 건물명, 호실, 임차인, 상태, 종료일, 남은 기간 표시
- 목록 항목 선택 시 계약 상세 화면으로 이동

다음과 같은 필터 목록 화면이 구현되어 있습니다.

- 오늘 후속 연락 계약 목록
- 협의 중 계약 목록
- 이번 달 만료 계약 목록
- 전체 계약 목록

### 2.3 계약 상세 화면

계약 1건의 상세 정보를 조회하고 즉시 관리할 수 있습니다.

표시 정보:

- 건물명
- 호실
- 임차인 이름
- 전화번호
- 계약 시작일
- 계약 종료일
- 남은 계약 기간
- 계약 상태
- 다음 연락일

상세 화면에서 가능한 작업:

- 임차인 전화 걸기
- 계약 안내 메시지 공유
- 다음 연락일 지정
- 다음 연락일 변경
- 다음 연락일 삭제
- 계약 상태를 협의 중으로 변경
- 계약 종료 처리
- 계약 2년 연장
- 계약 수정 화면으로 이동
- 계약 삭제

### 2.4 계약 등록/수정 화면

새 계약을 등록하거나 기존 계약을 수정할 수 있습니다.

입력 항목:

- 건물명
- 호실
- 임차인 이름
- 전화번호
- 계약 시작일
- 계약 종료일

검증 기능:

- 필수 입력값 검사
- 계약 시작일/종료일 미선택 시 등록 차단
- 종료일이 시작일보다 빠를 경우 등록 차단

등록 시 기본값:

- 상태는 `active`
- 다음 연락일은 없음

## 3. 데이터 관리 기능

### 3.1 계약 데이터 모델

계약은 다음 정보를 기준으로 관리됩니다.

- 고유 ID
- 건물명
- 호실
- 임차인 이름
- 전화번호
- 계약 시작일
- 계약 종료일
- 계약 상태
- 다음 연락일

계약 상태는 아래 3가지를 사용합니다.

- `active`
- `negotiating`
- `ended`

### 3.2 계약 CRUD

현재 구현된 데이터 조작 기능은 다음과 같습니다.

- 계약 생성
- 계약 조회
- 계약 수정
- 계약 삭제

### 3.3 정렬 및 집계

대시보드와 목록을 위해 다음 로직이 구현돼 있습니다.

- 계약 종료일 기준 정렬
- 오늘 날짜와 같은 다음 연락일 집계
- 상태가 `negotiating`인 계약 집계
- 현재 월에 종료되는 계약 집계
- 만료일이 가장 가까운 계약 상위 3건 추출

## 4. 저장 기능

계약 데이터는 로컬 저장 방식으로 유지됩니다.

- `SharedPreferences` 사용
- 앱 재실행 후에도 계약 데이터 유지
- 저장 키: `leases_v1`

저장된 데이터가 없을 경우 샘플 계약 데이터로 초기화됩니다.

초기 샘플 데이터 목적:

- 첫 실행 시 빈 화면 방지
- 기능 테스트용 기본 데이터 제공

## 5. 알림 기능

로컬 알림 기능이 구현돼 있습니다.

- 다음 연락일 오전 9시 알림
- 계약 만료 7일 전 오전 9시 알림
- 계약 만료 당일 오전 9시 알림

알림 동작 방식:

- 앱 시작 시 전체 계약 기준으로 알림 동기화
- 계약 등록 시 해당 계약 알림 생성
- 계약 수정 시 해당 계약 알림 갱신
- 계약 삭제 시 해당 계약 알림 취소
- 다음 연락일이 없거나 이미 지난 경우 후속 연락 알림 취소

## 6. 외부 연동 기능

현재 외부 패키지를 활용한 사용자 액션은 다음과 같습니다.

- `url_launcher`: 전화 앱 실행
- `share_plus`: 메시지 공유
- `flutter_local_notifications`: 로컬 알림 예약
- `flutter_timezone` + `timezone`: 로컬 시간대 기준 알림 예약

## 7. 현재 기준 미구현 항목

코드 기준으로 아직 구현되지 않은 항목은 다음과 같습니다.

- 사용자 로그인/권한 관리
- 서버 연동 또는 클라우드 동기화
- 검색 기능
- 복합 필터 기능
- 계약 메모/첨부파일
- 월세/보증금/납부 이력 관리
- 알림 설정 화면
- 통계 리포트 화면
- 다국어 처리

## 8. 참고 사항

- 현재 일부 UI 문자열은 한글 인코딩이 깨져 보일 가능성이 있습니다.
- 기능 로직은 존재하지만 화면 문구 품질은 별도 점검이 필요합니다.
